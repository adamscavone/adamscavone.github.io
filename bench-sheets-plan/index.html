Decision 1 of 6

Replace the manual SOP checklist with auto-validation?

Today: a popup with a Pass/Fail row per checklist item, primary + secondary review tabs. The chemist clicks through the list manually. Most items could be checked automatically from data the system already has (LCS in spec, blank clean, brackets closed, controls evaluated, etc.).

Proposed: drop the popup. Sign-off happens on the Run Validation page, which auto-grades the run and only asks the chemist about the gaps the system genuinely can't see (sample physical condition, exclusion rationale, conditioning OK, override justification, deviations).

Decision 2 of 6

Approve the 14 auto-checks the system will grade?

If we collapse the checklist, the auto-validator has to grade these things at sign-off. Listed in plain English. Items 13–14 already exist as the page's sign-off blocker; the rest are new.

Scientific QC
  1. LCS recovery is within the SOP-defined range.
  2. Method blank is clean — no target analyte above the reporting limit (or ½ RL per SOP).
  3. Matrix spike recovery is within limits.
  4. Duplicate precision (RPD) is within method maximum.
  5. CCV bracketing is complete — no more than the SOP-allowed customer injections between passing CCVs.
  6. Every control row has been evaluated (pass/fail recorded, not still null).
Audit trail
  1. Every injection in the run resolves to exactly one owner — either a control row or a customer sample, never both, never neither.
  2. Every released sample traces back to a bench sheet (originating prep batch in Ready or Sent state).
  3. Per-sample provenance is complete — every reportable sample has weigher, extractor, diluter, transferrer, and chemist recorded.
  4. Per-control provenance is complete — same audit fields recorded for every control row.
  5. Lot traceability — LCS, internal standard, mobile phase, and extraction solvent lots are recorded at the batch or row level.
  6. Every piece of equipment used (balance, pipette, repeater, column) was within its calibration window on the prep date.
Workflow gates (already exist)
  1. Every sample has a release disposition that's resolved (no stalled "awaiting close" or "blocked").
  2. No override requests are still pending at sign-off.

Phasing is OK — we can ship the highest-value 6–8 first and add the rest later if any of them aren't ready on day one.

Decision 3 of 6

Approve the 5 things the chemist will manually attest?

The flip side of auto-validation: things the system genuinely can't see and that need a human attestation. The form on the Run Validation page only shows prompts the system couldn't auto-derive — if a run has none of these conditions, the form is empty.

  1. Sample physical condition / handling anomaly — phase separation, clogging, viscosity, leakage, foaming. Pass/fail with required note on fail.
  2. Reason an injection was excluded or marked non-operative — the system records that an injection was excluded; only the chemist knows why (carryover, vial crash, autosampler miss, conditioning).
  3. Instrument conditioning was acceptable — no entity captures conditioning state. Chemist attests yes/no.
  4. Override justification narrative — system detects an override is needed; chemist writes the scientific rationale.
  5. Approved-deviation rationale — when the auto-validator flags a missing expected step or off-template parameter that the chemist deliberately did, the rationale gets recorded.
Decision 4 of 6

Where does the default replicate count come from?

"Add Replicates" needs to know how many replicates per sample to create by default. SOP-140 says 3 for cannabinoid potency. Where should that live?

Decision 5 of 6 · highest-stakes

What happens when data changes after a run is signed off?

Setup: chemist signs off a run on Monday. Tuesday, a late control file lands for the same batch. Or an override gets approved. Or someone fixes the originating bench sheet. The "Validated" stamp from Monday now sits on top of evidence that has changed since.

Codex flagged this as the single biggest audit risk in the plan. v1 didn't address it. We need a policy.

Decision 6 of 6

How do we prevent pencil-whipping on the Sign-Off tab?

The granular role model captures who weighed, who diluted, who transferred, who ran the instrument, who reviewed primary, who reviewed secondary. When one chemist did everything, asking them to click 5–6 separate buttons (all stamping their own ID) tempts them to "pencil-whip" the others to save time.

Gemini flagged this risk: granular roles without UI shortcuts will end up faked.
