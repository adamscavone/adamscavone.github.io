▮The Problem

NCTL generates Certificates of Analysis but currently relies on Confident Cannabis (an external SaaS) as the authoritative archive of every CoA ever issued. If they go down, change terms, or have a contract dispute, NCTL loses immediate access to evidence of work it did.

▶The Solution

Pull every of-record CoA out of Confident into NCTL's own Azure Files share, with SHA-256 integrity proof and per-file provenance. After this ships, NCTL's internal LIMS becomes the system of record — future amendments originate in LIMS (separate initiative) and get pushed to Confident.

✓What You Build

  • Standalone .NET 8 worker (Azure WebJob) — reads production NCLims read-only, calls Confident's API, downloads each CoA PDF, writes to existing Azure Files share.
  • Going-forward scanner — polls ReportFile hourly so newly-issued CoAs (including Pearson's micro release) land in the archive automatically.
  • Operational ledger (your own Azure SQL DB, 3 tables) — every imported file has SHA-256, provenance, status. Auditor-replayable.

✕What You DON'T Touch

  • NCLims source code — no edits. Kondo branch and Pearson's microbials release are unaffected.
  • Production NCLims schema — no migrations. Your ledger lives in its own DB.
  • Byte-by-byte hash comparison as the reconciliation primitive. Use Confident's per-file public_key Guid instead — hashes false-positive on rendering drift. (Detail in section 3.)
  • LIMS-side amendment UX — deferred to a follow-up initiative after Pearson ships.

⚠Key Gotchas

  • Azure Files, not Blob Storage. In Portal nav: Storage account → File shares, not Containers.
  • Lab slugs are NCTL (Ohio, LabId=1001) and NCTLM (Michigan, LabId=1003) — not oh/mi.
  • Filter ReportType = 2 when looking for CoAs (1=MetrcCsv, 4=ManifestReport, 5=ConsolidatedCoA).
  • Raw Azure Files URLs don't open in a browser — auth headers required. Use Portal Download button or SAS URL.
  • The existing CcApiService.cs has known bugs (no retries, broken modifiedSinceTime); see Section 5.

⊕Volume & Timing

  • ~200,000 historical samples across both labs.
  • ~400,000 network ops total (one Sample call + one PDF download per sample).
  • At 0.5 req/sec sustained (recommended): ~5–7 days wall-clock.
  • ~200 GiB of new bytes added to the share (currently 504 GiB; plenty of headroom).
  • Resumable per-sample — stop/restart freely, no data loss.
